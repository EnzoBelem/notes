
O Docker oferece uma interface de linha de comando poderosa para gerenciar containers, imagens e outros recursos. Abaixo, segue uma lista dos comandos mais usados, com explicações e exemplos.

---

## 1. Gerenciamento de imagens

### 1.1. docker pull

Faz o download de uma imagem do Docker Hub ou de um repositório específico.

**Sintaxe:**
``` bash
docker pull <nome_da_imagem>
```

### 1.2. docker images

Lista todas a imagens disponíveis localmente.

**Sintaxe:**
``` bash
docker images
```

**Saída:**
```bash
REPOSITORY   TAG       IMAGE ID       CREATED        SIZE
nginx        latest    abcd1234       2 days ago     25MB
```

### 1.3. docker rmi

Remove uma ou mais imagens.

**Sintaxe:**
```bash
docker rmi <image_id>
```

## 2. Gerenciamento de containers

### 2.1. docker run

Cria e inicia um novo container a partir de uma imagem.

**Sintaxe:**
```bash
docker run [opções] <imagem> [comando] [argumentos]
```

---
#### 1. Controle de execução

##### 1.1. `-d, --detach`

Inicia o container em segundo plano (modo "detached"). Ideal para aplicações que precisam rodar continuamente.

**Exemplo:**
```bash
docker run -d nginx
```

##### 1.2. `--rm`

Remove automaticamente o container quando ele for encerrado.

**Exemplo:**
```bash
docker run --rm ubuntu echo "Hello, World!"
```

##### 1.3. `--name`

Especifica um nome personalizado para o container (ao invés de um nome aleatório).

**Exemplo:**
```bash
docker run --name meu-container nginx
```

---
### 2.2. docker ps

Lista os containers em execução.

**Sintaxe:**
```bash
docker ps
```

Para listar **todos os containers**, inclusive os parados, use o flag `-a`:
```bash
docker ps -a
```

### 2.3. docker stop

Encerra um container em execução.

**Sintaxe:**
```bash
docker stop <container_id>
```

### 2.4. docker start

Reinicia um container previamente parado.

**Sintaxe:**
```bash
docker start <container_id>
```

### 2.5. docker rm

Remove um container.

**Sintaxe:**
```bash
docker rm <container_id>
```

## 3. Informações e Logs

### 3.1. docker logs

Exibe os logs de um container.

**Sintaxe:**
```bash
docker logs <container_id>
```

Para visualizar os logs em tempo real, use a flag `-f`:
```bash
docker logs -f <container_id>
```

### 3.2. docker inspect

Exibe informações detalhadas sobre um container ou imagem.

**Sintaxe:**
```bash
docker inspect <container_id|imagem>
```
